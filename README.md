# 疫情防控下外勤人员监测系统（前端）

一个基于 Vue 2 的前端项目，用于展示和管理外勤人员在疫情防控场景下的多维监测信息，包括人员信息、路径监测、体征监测、环境监测、设备信息与预警信息等模块。

---

## 一、项目背景

在疫情防控常态化管理中，外勤人员存在流动性高、工作环境复杂、健康状态变化快等特点。为了提升管理效率与风险响应能力，本项目实现了一个可视化前端系统，用于统一展示外勤人员相关数据，辅助管理人员进行快速查看、分析和预警处理。

本项目重点解决以下问题：

- 外勤人员信息分散、查询效率低；
- 路径、体征、环境等监测维度缺乏统一入口；
- 预警信息难以及时发现和处理；
- 业务页面需要具备清晰导航与模块化展示能力。

---

## 二、技术栈

### 核心框架

- **Vue 2.5**：前端核心框架
- **Vue Router 3.x**：前端路由管理
- **Element UI 2.x**：界面组件库
- **Vue Baidu Map**：地图能力集成（需百度地图 AK）

### 工程化与构建

- **Webpack 3**：项目构建与打包
- **Babel**：ES6+ 转译
- **Sass / CSS Loader**：样式处理
- **Webpack Dev Server**：本地开发调试

---

## 三、实现功能

根据当前路由与页面结构，项目包含以下主要功能模块：

- **首页（Front）**
  - 项目入口页
  - 提供登录跳转与基础导航

- **登录页（Login）**
  - 用户登录入口

- **主控页面（Main）及子模块**
  - **人员信息（people）**：外勤人员基础信息管理与查看
  - **路径监测（way）**：人员活动路径相关展示
  - **体征监测（sign）**：健康体征相关数据展示
  - **环境监测（environment）**：外部环境指标展示
  - **设备信息（equipment）**：监测设备信息展示
  - **预警信息（warning）**：异常预警信息展示与查看

- **分页组件（pagination）**
  - 提供分页交互能力

## 五、环境要求

建议使用：

- **Node.js** >= 14（项目模板较旧，若安装依赖报错可尝试 Node 14/16）
- **npm** >= 6

> 说明：项目最初模板的 engines 声明为较低版本（Node >= 6），但在现代环境下建议使用较新的 LTS 版本以保证工具链稳定性。

---

## 六、安装与运行

在项目根目录执行：

```bash
# 安装依赖
npm install

# 本地开发（默认 http://localhost:8080）
npm run dev

# 生产构建
npm run build
```

## 七、路由说明（简要）

主要路由入口：

- `/front`：首页
- `/login`：登录页
- `/main`：主控页（承载子路由）
  - `/people`
  - `/way`
  - `/sign`
  - `/environment`
  - `/equipment`
  - `/warning`

---

## 八、地图能力说明

项目使用 `vue-baidu-map`，需要在路由初始化处配置百度地图 AK。

如果你要在自己的环境部署，请替换为你自己的 AK，避免使用无权限或已失效密钥。
